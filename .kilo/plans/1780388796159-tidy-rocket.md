# Plan: Add Supplier Management Page

## Goal
Implement the supplier management admin page for the PhoneFix Shop admin panel, with:
- **GET /admin/providers** listing all suppliers
- **PATCH /admin/providers/{provider_id}?is_active=<bool>** to toggle a supplier active/inactive

This allows an admin to disable/enable suppliers so their listings are included or excluded from public `GET /items`.

## Changes Required

### 1. `src/services/api.js`
Add two new exported functions:
- `getProviders()` → `api.get("/admin/providers")`
- `toggleProviderStatus(providerId, isActive)` → `api.patch(`/admin/providers/${providerId}?is_active=${isActive}`)`

### 2. `src/pages/Providers.jsx` (new file)
Create a new page component modeled after `Users.jsx` but for suppliers:
- State: `providers`, `loading`, `error`
- `useEffect` to call `getProviders()`
- `toggleProviderStatus` updates local provider state after API call
- Table columns: **Name**, **Active**, **Base URL**, **Notes**
- Action column with a toggle button (Active / Inactive) per row
- Use existing `Card`, `Table`, `Button` components

### 3. `src/App.jsx`
Add a new route:
- `/suppliers` → wrapped in `ProtectedRoute` and `DashboardLayout` with `<Suppliers />`

### 4. `src/components/Sidebar.jsx`
Add a new sidebar nav entry under **Catalog** (or a new **Suppliers** section):
- `{ path: '/suppliers', icon: '🏭', label: 'Suppliers' }`

## Assumptions

- No pagination, search, or filter is requested in the spec.
- Error handling follows the existing pattern (simple `catch` setting an error message string).

## What this page actually does

The backend only exposes:
- `GET /admin/providers` → list all suppliers
- `PATCH /admin/providers/{provider_id}?is_active=<bool>` → toggle active/inactive

There is **no** create/edit/delete supplier endpoint documented. Suppliers are managed here by quickly toggling `is_active`. This maps directly to the public `GET /items` behavior: when a supplier is inactive, all its `ProviderListing` rows are excluded from public results, so items backed only by that supplier drop out of the listings.

## Page scope
- Table of suppliers: Name, Base URL, Notes, Active status
- One action per row: toggle Active/Inactive button
- No create/edit/delete forms (no API support)
- No bulk actions, search, or pagination (not requested)
