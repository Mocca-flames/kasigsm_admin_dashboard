# Plan: Implement Banners Page

## Overview
Add a new Banners admin page to design, create, edit, and delete banners per the admin_endpoints.md specification.

## Tasks

### 1. Add API functions (`src/services/api.js`)
- `getBanners()` - GET /admin/banners
- `createBanner(bannerData)` - POST /admin/banners
- `updateBanner(bannerId, bannerData)` - PATCH /admin/banners/{banner_id}
- `deleteBanner(bannerId)` - DELETE /admin/banners/{banner_id}
- `toggleBannerActive(bannerId, isActive)` - PATCH /admin/banners/{banner_id}/toggle

### 2. Create Banners page component (`src/pages/Banners.jsx`)
- State management for banners list, loading, error, modal
- Load banners on mount
- Create/Edit modal with form fields matching API spec:
  - slug (required)
  - title (required)
  - content (required)
  - image_url (optional)
  - link_url (optional)
  - is_active (default: true)
  - is_dismissible (default: true)
  - starts_at (optional datetime)
  - ends_at (optional datetime)
- Table display with columns: title, content, is_active, actions
- Delete confirmation dialog
- Toggle active status

### 3. Update Sidebar (`src/components/Sidebar.jsx`)
- Add Banners nav item under "Tools" section
- Icon: 📢 (megaphone), label: "Banners"

### 4. Update App routes (`src/App.jsx`)
- Import Banners component
- Add route: `/banners`
