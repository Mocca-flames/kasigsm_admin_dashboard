# Plan: Admin Thumbnail & Image Management via Drag-and-Drop

## Objective
Replace all URL-paste image fields in the admin dashboard with a consistent, reusable drag-and-drop upload experience. Keep admin flow unambiguous by centralising image handling into one component and wiring it to the three existing backend upload endpoints.

## Assumed Backend Endpoints (already live)
- `POST /admin/upload` → `multipart/form-data` with `file` field → `{url: "/media/..."}`
- `PATCH /admin/items/{id}/thumbnail` → same file input
- `PATCH /admin/providers/{id}/logo` → same file input

---

## 1. Add Reusable DropZone Component
**File:** `src/components/DropZone.jsx` (new)

Props:
- `onFileSelected(file)`: called after a successful upload, returns the resulting URL
- `value` (string, optional): existing image URL → shown as preview
- `onClear` (optional): clears current image
- `accept` (string, default `"image/*"`): accepted MIME types
- `disabled` (bool)

Behaviour:
- Drag-and-drop zone + click-to-browse fallback via hidden `<input type="file">`
- On drop / select → `POST /admin/upload` via FormData → populates preview
- If `value` prop is set, renders a small thumbnail preview with an ✕ remove button
- Loading/uploading state with spinner
- Error state if upload fails
- Area clearly labels "Drop image here or click to browse" so admin intent is obvious
